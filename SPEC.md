# WanderPlan — Full Product Specification & Fable Build Prompt

---

## Project Overview

Build a full-stack AI-powered travel planning web application called **WanderPlan**. Users can select up to 5 countries on an interactive 3D globe, set trip dates and start/end cities, define their travel preferences via sliders, and receive a fully personalized, day-by-day itinerary generated by AI. The app supports user auth, collaborative trip planning with friends, and live event discovery via web scraping.

## Feature Specification

### 1. Interactive Globe (Home Screen)
- Full-screen 3D globe using `react-globe.gl`
- Users click or search to highlight up to 5 countries (highlighted in electric teal accent color when selected)
- Selected countries appear as a chip stack at the bottom of the screen
- Globe rotates gently on idle; snaps/pans to selected countries on click
- Country tooltip on hover shows: flag, name, avg daily cost, best season, safety score
- "Plan This Trip" CTA button activates once at least 1 country is selected

### 2. Trip Setup Flow (Multi-step wizard)

**Step 1 — Dates & Locations**
- Date range picker (start date to end date), min 3 days, max 365 days
- Start city input (autocomplete via Google Places API)
- End city input (can be same as start — "return trip" toggle)
- Total trip duration shown dynamically

**Step 2 — Travelers**
- Number of travelers (1-20)
- Ages/group type selector: Solo / Couple / Friends / Family with Kids
- Dietary restrictions (multi-select: vegan, halal, kosher, gluten-free, etc.)
- Mobility/accessibility needs toggle

**Step 3 — Preferences Sliders Panel**
16 sliders, each 0-100, grouped into 4 color-coded categories:

Experiences (green):
- Food & Food Festivals
- Nature, Hikes & Waterfalls
- City Life & Local Events
- Music & Concerts
- Beaches & Swimming
- Wildlife & National Parks
- Photography & Scenic Spots
- Nightlife & Night Markets
- Wellness & Retreats
- Culture, Museums & Heritage

Pace & Style (blue):
- Trip Pace (Relaxed to Packed)
- Discovery Style (Tourist Hotspots to Hidden Gems)

Practical (amber):
- Budget Sensitivity (Splurge to Minimize Spend)
- Flight Aversion (Fine with flying to Avoid flights)

Accommodation (purple):
- Accommodation Style (Backpacker to Luxury)
- Accommodation Type (Hotel to Airbnb/Local stays)

**Step 4 — Budget**
- Daily budget per person in USD (or local currency toggle)
- Optional: total trip budget cap

**Step 5 — Review & Generate**
- Summary card of all selections
- "Generate My Itinerary" button triggers AI call with streaming

### 3. AI Itinerary Generation
- System prompt includes: country context from DB (top experiences, avg costs, events), user preferences, dates, group size, dietary needs
- Claude generates structured JSON: array of day objects with morning/afternoon/evening activities, transport notes, accommodation suggestion, estimated cost, local tips
- Response is streamed to UI (typing effect per day card)
- Each day card shows a mini map pin for the day's location
- Users can "Regenerate Day" for any single day without touching the rest
- Users can "Lock" days they're happy with before regenerating others

### 4. Live Event Discovery (Web Scraper)
- Puppeteer + Cheerio scraper runs on a daily cron job per country
- Sources: Eventbrite (country pages), Resident Advisor (music), local tourism board sites, Google Events API
- Scraped events cached in Redis with 24h TTL and written to live_events table
- AI prompt includes upcoming events during trip dates for each country
- "Live Event" badge appears on itinerary days when a scraped event matches the day

### 5. User Auth & Accounts
- NextAuth with Google, GitHub, and Magic Link (email)
- Profile page: home country, passport country, travel history, saved trips
- Saved trips dashboard with status: Draft / Planning / Confirmed / Completed

### 6. Collaborative Trip Planning
- "Invite Friends" button generates a shareable link (UUID token)
- Invited users join as Editor (can edit preferences/itinerary) or Viewer
- Real-time collaboration via Supabase Realtime (presence indicators, live updates)
- Per-day comments/chat on itinerary cards (threaded)

### 7. Accommodation Integration
- Each itinerary day surfaces 2-3 accommodation options pulled from DB
- Deep-link to Airbnb listing or hotel booking page
- Filter by type: Hotel / Hostel / Airbnb / Resort
- Price shown per night per person

### 8. Map View
- Toggle between Day View (card stack) and Map View (full Mapbox map with route drawn between all day locations)
- Route line animated, color-coded by transport type: flight=dashed, train=solid blue, bus=solid orange

## UI/UX Design Direction

**Visual Identity:**
- Dark navy base (#0A0F1E) — night-sky/globe aesthetic
- Primary accent: electric teal (#00E5C3) for selected countries and CTAs
- Secondary accent: warm amber (#F5A623) for budget/practical sliders
- Globe rendered with subtle country borders, glowing pulse on selected countries
- Typography: Syne (display, geometric) paired with Inter (body/UI)
- Glassmorphism cards for day itinerary items (frosted dark glass over map/photo backgrounds)
- Custom slider thumbs: pill shape with emoji icon matching the category

**Signature Element:**
The globe persists as a glowing miniature in the top-right corner of all planning and itinerary views, with selected countries still lit up — users always feel geo-anchored to their trip regardless of which screen they're on.

## Web Scraper Architecture

Scraper runs as a separate Node.js process on a daily cron:

- `scraper/index.ts` — orchestrator, loops through active countries
- `scraper/sources/eventbrite.ts` — scrapes Eventbrite country pages
- `scraper/sources/residentadvisor.ts` — music/club events per city
- `scraper/sources/tourismboards.ts` — per-country tourism board event pages (configurable URL list in DB)
- `scraper/sources/googleevents.ts` — Google Events API where available

Rate limiting: max 1 req/sec per domain, randomized user-agent rotation.
Deduplication: by (name + start_date + city) composite key before DB insert.
Redis key pattern: `events:{country_code}:{YYYY-MM}` with 24h TTL.

## Phased Rollout

**Phase 1 — Core MVP**
Globe + country selection, trip wizard with sliders, AI itinerary generation, Google OAuth, basic Postgres schema seeded for 50 countries, day-by-day itinerary view.

**Phase 2 — Live & Social**
Web scraper (Eventbrite + Resident Advisor), collaborative trips (invite links, editor/viewer roles), full map view with animated route, Airbnb/hotel deep links.

**Phase 3 — Polish & Scale**
Per-day chat/comments, regenerate single day, lock days, PWA/mobile, expand to all 195 countries, flight search integration (Skyscanner API), budget tracker (actual vs. estimated spend).
